Explanation

<ProductFormFeatureDescription> contains a short addendum to the accessibility detail of the product, or the URL of a web page comprising such an addendum, providing further details of the specific conformance and features derails provided. The addendum should be considered additional to the more structured data. A complete Accessibility summary (including information that is duplicated in the structured data) should be provided using code 00. Code 92 is used to provide a summary of the accessibility.

The summary provided in this field corresponds to the accessibility summary as defined in the EPUB Accessibility 1.1 specification. That is, it does not duplicate information such as the conformance statement and accessibility features of the publication.

Note

For publications that conform to the EPUB Accessibility 1.0 specification — where the summary may reproduce information expressed in the other accessibility metadata — use Code 00 instead.

The value of this field is the same as would be expressed in the Schema.org accessibilitySummary property.

The field is often limited to only 500 characters.
