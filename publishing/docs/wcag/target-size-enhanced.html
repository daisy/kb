About

Version: WCAG 2.1 and up
Number: 2.5.5
Level: AAA
Applicability:
  • Interactive Content
  • Links
Requirement:

The size of the target for pointer inputs is at least 44 by 44 CSS pixels except when:

Equivalent

The target is available through an equivalent link or control on the same page that is at least 44 by 44 CSS pixels;

Inline

The target is in a sentence or block of text;

User Agent Control

The size of the target is determined by the user agent and is not modified by the author;

Essential

A particular presentation of the target is essential to the information being conveyed.
Guidance:
  • Understanding Success Criterion 2.5.5
  • How to meet Success Criterion 2.5.5
Techniques: WCAG currently only includes the following undocumented techniques for this success criterion:
  • Ensuring that touch targets are at least 44 by 44 CSS pixels.
  • Providing a mechanism to change the size of the target independent of magnification.

Purpose

The purpose of this success criterion is to aid users who lack the ability or dexterity to activate small targets with a pointer device.

It differs from 2.5.8 primarily in having a larger minimum size for the target area. The exception for padding a target to meet the minimum target area size also does not apply.

How to Meet

Except for the cases listed in the success criterion text, authors need to ensure that all target areas (buttons, etc.) are at least 44 pixels by 44 pixels in dimensions
