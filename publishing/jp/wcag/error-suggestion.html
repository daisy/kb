About

Version: WCAG 2.0 and up
Number: 3.3.3
Level: AA
Applicability:
  • Form Inputs
Requirement:

If an input error is automatically detected and suggestions for correction are known, then the suggestions are provided to the user, unless it would jeopardize the security or purpose of the content.
Guidance:
  • Understanding Success Criterion 3.3.3
  • How to meet Success Criterion 3.3.3
Techniques:
  • ARIA2: Identifying a required field with the aria-required property
  • ARIA18: Using aria-alertdialog to Identify Errors
  • G83: Providing text descriptions to identify required fields that were not completed
  • G84: Providing a text description when the user provides information that is not in the list of allowed values
  • G85: Providing a text description when user input falls outside the required format or values
  • G177: Providing suggested correction text
  • SCR18: Providing client-side validation and alert
  • SCR32: Providing client-side validation and adding error text via the DOM

Purpose

The purpose of this success criterion is to help user understand how to fix input errors.

How to Meet

Authors need to provide guidance on how to correct input errors whenever it is possible. If an input field has a limited set of acceptable values, for example, the range can be listed or described. Or if a user has written too long a response for a field, the size limit could be noted as well as how much text has been input.
